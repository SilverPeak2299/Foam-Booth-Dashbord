# Data Sources And Privacy Rules

## Public Repo Policy

This repository is public and the dashboard is intended for public GitHub Pages deployment through GitHub Actions.

No PII should be committed to this repo or emitted into the built static site.

Blocked from this repo and from deployed artifacts:

- Raw customer names.
- Raw free-text descriptions.
- Raw sales exports containing customer names or descriptions.
- Reversible customer mapping tables.
- Banking/account details from descriptions.
- Any source file that has not passed a sanitization step.

## Known Source Data

Current upstream analysis repo:

- Raw export: `../Foam-Booth-Data-Analysis/data/data.txt`
- Semi-prepared CSV: `../Foam-Booth-Data-Analysis/data/stage_1_output.txt`

These files must be treated as private inputs because they include customer names and raw descriptions.

Known shape of `stage_1_output.txt`:

- `Co./Last Name`
- `Date`
- `Invoice #`
- `Item Number`
- `Quantity`
- `Description`
- `Price`
- `Total`

Known data coverage:

- 83,107 prepared line items.
- 39,231 unique invoices.
- Date range: 2006-11-09 to 2025-05-02.

## Allowed Public Artifacts

Public dashboard artifacts should be aggregate-only JSON files, such as:

- `summary.json`
- `monthly_sales.json`
- `yearly_sales.json`
- `category_trends.json`
- `top_items.json`
- `top_foam_types.json`
- `foam_seasonality.json`
- `top_fabrics.json`
- `fabric_rankings.json`
- `fabric_environment.json`
- `data_quality.json`
- `source_manifest.json`

Allowed public fields:

- Dates aggregated to day, month, season, or year.
- Invoice counts.
- Line counts.
- Revenue totals.
- Product categories.
- Foam codes.
- Foam thicknesses.
- Indoor/outdoor classification and confidence.
- Fabric names/brands only after review confirms they are product names, not customer names.
- Fabric indoor/outdoor-suitable inference and confidence.
- Data quality counts.

Blocked public fields:

- `Co./Last Name`
- `First Name`
- raw `Description`
- raw `Invoice #` when paired with row-level detail
- customer hash if it can be linked back to individual behavior in public output

## Sanitized Source Option

If a source file is committed to this repo later, it must be generated by a sanitization script and should contain only fields needed for public aggregate analytics.

Required sanitization:

- Drop all customer name fields.
- Drop raw descriptions after extracting safe structured fields.
- Drop or aggregate invoice IDs unless row-level invoice analysis is explicitly needed.
- Remove banking/account details.
- Run automated privacy checks before commit.

Customer IDs should not be included in v1. If repeat-customer analytics are added later, use a keyed HMAC generated with a GitHub Actions secret. Do not commit the key, salt, or mapping table.

## Revenue Policy

Dashboard headline revenue should default to product/service revenue.

Separate reconciliation views should show:

- discounts
- delivery/freight
- refunds or negative lines
- zero-dollar rows
- banking/non-product rows
- unknown/miscellaneous rows

## Foam Analytics Priority

The first foam analytics pass should produce:

- foam code ranking by revenue and line count.
- foam thickness ranking by revenue and line count.
- foam popularity by month, season, and year.
- indoor vs outdoor foam split.
- confidence and unknown counts.

## Fabric Analytics Priority

The first fabric analytics pass should produce:

- fabric ranking by revenue and estimated meters.
- fabric brand/name ranking.
- fabric trends by month/season/year.
- inferred fabric environment: `outdoor_suitable`, `indoor`, or `unknown`.
- confidence and reasoning counts.

Fabric indoor/outdoor classification is an inference, not a fact:

- Known outdoor-suitable brands/terms such as Sunbrella and Dickson can be high-confidence signals.
- Explicit outdoor wording in a fabric description can be medium-to-high confidence.
- Same-invoice association with outdoor foam is only low confidence because customers may use outdoor fabrics indoors.
